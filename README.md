# 🧪 Automation Exercise — QA Testing Project

A comprehensive Quality Assurance project covering **Manual Testing**, **API Testing**, **Test Automation**, and **Database Testing** for the [AutomationExercise](https://automationexercise.com) web application, with project management tracked via Jira.


## 📌 Project Overview

This project was developed as a full-cycle QA exercise, simulating a real-world testing workflow. It includes:

- ✅ Manual UI test cases covering core user flows
- ✅ API testing for key endpoints
- ✅ Automated test scripts using **Selenium + Java + TestNG**
- ✅ Database validation queries using the **Sakila** sample database
- ✅ Test planning and defect tracking via **Jira**

## 🧩 Testing Scope

### 🖱️ Manual Testing
- Test cases for UI flows: Registration, Login, Product Search, Cart, Checkout, Contact Form
- Covered positive, negative, and edge cases
- Linked to Jira user stories and tracked for pass/fail status

### 🔌 API Testing
- Validated REST API endpoints provided by the site
- Verified status codes, response bodies, and data integrity

### 🤖 Automated Testing (Selenium + Java + TestNG)
- Page Object Model (POM) design pattern
- Data-driven test execution
- Assertions and reporting via TestNG

### 🗄️ Database Testing (Sakila DB)
- SQL queries to validate data at the database layer
- Covered SELECT, JOIN, aggregation, and filter scenarios
